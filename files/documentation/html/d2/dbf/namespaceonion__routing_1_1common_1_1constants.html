◆ MY_SOCKS_SIGNATURE

int onion_routing.common.constants.MY_SOCKS_SIGNATURE = 0x80

Private method for socks5 protocol.

Special self recognition method for deciding whether to decrypt recieved data in node_server.pollables.socks5_server. Data should not be decrypted if node acts as proxy, unless its the last node. In case of the last node the missing signature is recognized and thus the server knows it should decrypt the encrypted messages.

◆ SOCKS5_STATES

tuple onion_routing.common.constants.SOCKS5_STATES = range(9)

Socks5 socket states for node_client.pollables.socks5_client and node_server.pollables.socks5_server.

  • RECV_GREETING: Recieve socks5 greeting from client.
  • SEND_GREETING: Send response for socks5 greeting to client.
  • RECV_CONNECTION_REQUEST: Recieve socks5 connection request from client.
  • SEND_CONNECTION_REQUEST: Send socks5 connection response to client.
  • PARTNER_STATE: Partner state when socket is used as TCP proxy after the end socks5 communication.
  • CLIENT_SEND_GREETING: Send socks5 greeting to server.
  • CLIENT_RECV_GREETING: Recieve socks5 greeting response from server.
  • CLIENT_SEND_CONNECTION_REQUEST: Send socks5 connection request to server.
  • CLIENT_RECV_CONNECTION_REQUEST: Recieve socks5 connection response from server.
