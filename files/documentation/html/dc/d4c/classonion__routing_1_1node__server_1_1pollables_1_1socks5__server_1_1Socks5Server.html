Detailed Description

Socks5 Server.

Created for routing a message from a client to a wanted destination. Uses rfc 1928 - https://www.ietf.org/rfc/rfc1928.txt. Supports only part of the options available in the protocol which are used for the purpose of the project.

On read event.

Read from _partner until maximum size of _buffer is recived. Run the current state.

_last_node is False -> decrypt content with key unless state is PARTNER_STATE. _last_node == True -> decrypt content with key at all times.
